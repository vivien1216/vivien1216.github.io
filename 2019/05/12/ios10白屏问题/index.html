ios10白屏问题

发表于 2019-05-12 | 分类于 Vue | | 次阅读

近日在负责公司的一个移动端项目，这段时间陆续有几个活动开始上线。突然被同事告知手机打不开页面，打开就是白屏。吓得我赶紧打开自己的手机看看，嗯，是好的呀？？？借用其他同事的手机一看，也是好的呀？？！！后来才发现，原来她的手机是ios10系统。在网上查询过后，造成ios10白屏问题还挺多的，嗯，在没有测试机的情况下，参考参考网上的建议看看吧。

一般来说，对于手机低版本系统，可能的白屏问题第一反应就是会不会是不兼容es6呢？于是,看了一下整个项目，先前的开发者已经配置好了es6的兼容问题，排除这个原因。结合项目看了其他几个可能造成白屏的问题，试着去解决，然而还是失败。而且，有时候按照网络上的解决办法来看，不一定能解决问题，还会带来其他一些麻烦。比如很多建议就是将config文件夹中的index.js中assetsPublicPath: '/'改为assetsPublicPath: './',然而我在使用这个解决办法后发现微信支付这块除了问题，后来发现是因为文件路径的问题导致微信支付出错，于是我又改了回来，微信支付这块恢复正常。在尝试了网上给的很多建议后，问题依旧没有解决。。。询问了一下ios开发同事，iOS10使用率大概为20%左右，还是挺大的一个比例，这个问题必须解决。
最后还是借来同事的手机进行真机调试，发现确实报了一个错，报错attempted to assign to readonly property，查询得知。这个报错是一个 ios中webkit内核的bug（而且每个框架都以独特的方式遇到这个问题).虽然知道了大概是因为什么问题导致白屏问题，但是具体还不知道到底是哪行代码导致的问题。问了一下使用iOS10系统的同事，得知之前手机是可以打开项目的，后来一段时间才打不开的。有了这个信息，那么就来查找到底是从哪个版本开始出了问题，然后看一下该版本修改了哪些代码就可以解决问题了。于是，使用git checkout + 版本号复制之前的版本，再使用git checkout -b + 分支名创建分支并切换分支，就可以将以前的老版本代码跑起来了。最后，很轻松的找到出错的版本，结合报错信息，尝试将可能导致白屏问题的代码注释后发现，嗯，白屏问题解决了。原来ios10系统window.pageYOffset = 0不支持这个。找到问题后就很容易解决了。开心～～～
嗯，之所以记录下这个问题，主要还是想记录下解决问题的一些过程和一些思考。有bug还是不能慌，一步一步解决才是最重要的，找到问题所在才是关键。前端路漫漫，加油！！！
