记录微信分享遇到的一些问题

发表于 2019-05-25 | 分类于 移动开发 | | 次阅读

这段时间一直在做移动端的一些活动，活动嘛，当然主要还是通过微信来进行推广的。所以，免不了涉及到微信分享这一块。今天就来总结一下我在做微信分享这一块遇到的各种问题。

使用weixin-js-sdk实现自定义分享

因为我用的是vue，所以直接引入weixin-js-sdk，通过config接口注入权限验证配置，分享失败时，可以将debug改为true,开启调试模式,调用的所有api的返回值会在客户端alert出来，若要查看传入的参数，可以在pc端打开，参数信息会通过log打出，仅在pc端时才会打印。如果分享失败可以看到是前端问题还是接口数据问题。其他一些具体操作可以在微信开发者工具中看到。地址：https://mp.weixin.qq.com/wiki?t=resource/res_main&id=mp1421141115。

ios中分享失败

之前在单独一个页面中做过微信分享，对这一块还是很有信心的。然而，测试跑过来跟我讲有问题，分享的页面链接不对，而且自定义标题和图片也都是错误的。我用我的andriod测试，嗯，再一次没问题呀。用了测试的ios测试，确实有问题。复制分享出去的链接看，发现虽然我页面跳转了，但是链接却依然是之前页面的链接，这是怎么回事？因为分享地址是后端重定向的，我还跑过去问了一下后端，结果被怼了。于是静下心来找问题。后来想起之前有同事跟我说过这个问题，好像ios不支持this.$router.push()。于是我尝试将this.$router.push()改为location.href,测试发现问题解决了。嗯，这段时间开发移动端发现，ios最容易出bug，很多东西可能就不支持。不过对于开发者来说，有bug还是要努力想办法解决。
